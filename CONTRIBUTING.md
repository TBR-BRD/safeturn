# Mitwirken

SafeTurn ist aktuell ein technischer Prototyp. Beitraege sind willkommen, aber bitte beachte:

1. Keine Funktionen als produktiv oder sicherheitszertifiziert darstellen.
2. Tests nur auf abgesperrtem Gelaende durchfuehren.
3. Keine personenbezogenen Daten in Beispielen oder Logs einchecken.
4. Aenderungen am BLE-Protokoll immer in `protocol/PROTOCOL.md` dokumentieren.
5. Aenderungen am Risikoalgorithmus immer mit Testfall beschreiben.

## Branches

Empfohlen:

```text
main                stabile Dokumentation / letzter MVP
feature/...         neue Funktionen
experiment/...      Hardware- und Algorithmus-Experimente
```
